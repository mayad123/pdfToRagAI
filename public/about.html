About this project

pdf-to-rag is a small, auditable pipeline: PDFs to text, chunks, embeddings, and a JSON vector store—with multiple ways to drive the same logic.

What ships

  • Command line — ingest folders, run queries, inspect indexes from a terminal.
  • Library — embed the same functions in your own Node.js tooling.
  • Model Context Protocol (MCP) — expose ingest, query, and inspect to editors and agents (stdio or HTTP).

Local-first and safety

Default embeddings use Transformers.js on your CPU; optional Ollama backends are supported. The server only reads paths you allow via environment configuration—there is no silent upload of your PDFs to a third party when you run it yourself.

This website

The Try demo page is a vanilla HTML client: it loads the official MCP TypeScript SDK and talks to /mcp on the same host as these pages. Starting npm run mcp:http in a terminal launches that host—there is no separate REST API layer for the demo.

Stack highlights

  • TypeScript
  • Node.js 18+
  • MCP
  • Transformers.js
  • PDF extraction & chunking
