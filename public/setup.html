Set up pdf-to-rag

You need Node.js 18 or newer and a copy of the project. One terminal command starts both the web pages and the engine the demo talks to.

  1. Get the code

    Clone the repository or unpack the npm package, then open a terminal in the project root.

    git clone <your-repo-url> pdf-to-rag
cd pdf-to-rag

  2. Install and build

    Install dependencies and compile TypeScript. The first embedding run may download a model (~tens of MB).

    npm install
npm run build

  3. Start the web server

    This serves the UI at http://127.0.0.1:3000/ (by default) and exposes the MCP endpoint at /mcp on the same port—so the browser and your PDF index stay on your machine. 

    npm run mcp:http
# → http://127.0.0.1:3000/  ·  MCP http://127.0.0.1:3000/mcp

    Use another port with PDF_TO_RAG_HTTP_PORT=4000 npm run mcp:http.

  4. Tell the server which folders are allowed

    For security, only certain paths can be read. If your PDFs live inside the project (e.g. examples/), defaults often work. For other layouts, set environment variables before starting the server—see the docs linked below.

    Common variables: PDF_TO_RAG_CWD (workspace root), PDF_TO_RAG_ALLOWED_DIRS (comma separated), PDF_TO_RAG_SOURCE_DIR (default corpus).

  5. Optional: use Cursor or another MCP host (stdio)

    Instead of the browser demo, you can register the built server in your editor so agents call ingest, query, and inspect directly. Replace paths with absolutes on your computer. 

    {
  "mcpServers": {
    "pdf-to-rag": {
      "command": "node",
      "args": ["/absolute/path/to/pdfToRag/dist/mcp/server.js"],
      "env": {
        "PDF_TO_RAG_CWD": "/absolute/path/to/pdfToRag",
        "PDF_TO_RAG_SOURCE_DIR": "/absolute/path/to/pdfToRag/examples",
        "PDF_TO_RAG_ALLOWED_DIRS": "/absolute/path/to/pdfToRag/examples"
      }
    }
  }
}
